# 合约市场结构评分模型

## 一、模型目标

不要试图精确计算：

```text
真实主力净流入
```

因为币安公开 API 无法提供：

```text
账户身份
大户真实成交金额
大户真实持仓金额
每笔成交是开仓还是平仓
```

所以模型目标改为：

```text
判断当前市场结构更适合做多、做空，还是观望
```

最终输出：

```text
趋势方向
动量方向
仓位结构
情绪状态
风险等级
交易信号
```

---

## 二、模型分层

模型分为 5 层：

```text
1. 趋势层
2. 动量层
3. 仓位层
4. 情绪层
5. 风险层
```

## 四、基础计算字段

### 1. 周期主动卖出额

```text
周期主动卖出额 =
    周期成交额 - 周期主动买入额
```

---

### 2. 主动买卖压力金额

```text
主动买卖压力金额 =
    周期主动买入额 - 周期主动卖出额
```

等价于：

```text
主动买卖压力金额 =
    2 × 周期主动买入额 - 周期成交额
```

解释：

```text
主动买卖压力金额 > 0：主动买盘更强
主动买卖压力金额 < 0：主动卖盘更强
主动买卖压力金额 ≈ 0：主动买卖接近平衡
```

---

### 3. 主动买卖压力比例

```text
主动买卖压力比例 =
    主动买卖压力金额 ÷ 周期成交额
```

判断：

```text
主动买卖压力比例 > 10%：主动买盘强
主动买卖压力比例 < -10%：主动卖盘强
-10% 到 10%：主动方向不明显
```

---

### 4. 持仓价值变化额

```text
持仓价值变化额 =
    当前总持仓价值 - 上一周期总持仓价值
```

解释：

```text
持仓价值变化额 > 0：总持仓增加
持仓价值变化额 < 0：总持仓减少
持仓价值变化额 ≈ 0：总持仓近似不变
```

---

### 5. 持仓变化比例

```text
持仓变化比例 =
    |持仓价值变化额| ÷ 周期成交额
```

判断：

```text
持仓变化比例 < 5%：持仓近似不变，偏换手
持仓变化比例 >= 5%：持仓明显变化
```

---

### 6. 成交额放大倍数

```text
成交额放大倍数 =
    当前周期成交额 ÷ 最近 N 个周期平均成交额
```

建议：

```text
5m 周期：N = 288，代表最近 24 小时
15m 周期：N = 96，代表最近 24 小时
1h 周期：N = 24，代表最近 24 小时
```

判断：

```text
成交额放大倍数 >= 1.2：成交放大
成交额放大倍数 < 1.2：成交未明显放大
```

---

### 7. 价格涨跌幅

```text
价格涨跌幅 =
    (当前收盘价 - 上一周期收盘价)
    ÷
    上一周期收盘价
```

判断：

```text
价格涨跌幅 > 0：上涨
价格涨跌幅 < 0：下跌
价格涨跌幅 ≈ 0：横盘
```

## 五、趋势层

### 1. 计算指标

```text
EMA20
EMA60
```

---

### 2. 趋势判断

```text
如果 当前价格 > EMA20 且 EMA20 > EMA60：
    趋势方向 = 多头趋势

如果 当前价格 < EMA20 且 EMA20 < EMA60：
    趋势方向 = 空头趋势

否则：
    趋势方向 = 震荡
```

---

### 3. 趋势分

```text
如果 当前价格 > EMA20 且 EMA20 > EMA60：
    趋势分 = +30

如果 当前价格 < EMA20 且 EMA20 < EMA60：
    趋势分 = -30

否则：
    趋势分 = 0
```

---

## 六、动量层

### 1. 动量判断

```text
如果 主动买卖压力比例 > 10%
并且 成交额放大倍数 >= 1.2：
    动量方向 = 多

如果 主动买卖压力比例 < -10%
并且 成交额放大倍数 >= 1.2：
    动量方向 = 空

否则：
    动量方向 = 弱
```

---

### 2. 动量分

```text
如果 主动买卖压力比例 > 10%
并且 成交额放大倍数 >= 1.2：
    动量分 = +25

如果 主动买卖压力比例 < -10%
并且 成交额放大倍数 >= 1.2：
    动量分 = -25

否则：
    动量分 = 0
```

---

## 七、仓位层

### 1. 仓位结构判断

```text
如果 价格上涨 且 持仓价值变化额 > 0：
    仓位结构 = 多头开仓推动

如果 价格上涨 且 持仓价值变化额 < 0：
    仓位结构 = 空头回补推动

如果 价格下跌 且 持仓价值变化额 > 0：
    仓位结构 = 空头开仓推动

如果 价格下跌 且 持仓价值变化额 < 0：
    仓位结构 = 多头平仓/止损

如果 价格横盘 且 持仓价值变化额 > 0：
    仓位结构 = 蓄势增仓

如果 价格横盘 且 持仓价值变化额 < 0：
    仓位结构 = 降温减仓

如果 持仓变化比例 < 5%：
    仓位结构 = 换手
```

---

### 2. 仓位分

```text
如果 价格上涨 且 持仓价值变化额 > 0：
    仓位分 = +25

如果 价格下跌 且 持仓价值变化额 > 0：
    仓位分 = -25

如果 价格上涨 且 持仓价值变化额 < 0：
    仓位分 = +10

如果 价格下跌 且 持仓价值变化额 < 0：
    仓位分 = -10

否则：
    仓位分 = 0
```

解释：

```text
价格上涨 + OI 增加：多头开仓推动，趋势持续性更强
价格上涨 + OI 减少：空头回补推动，持续性弱一些
价格下跌 + OI 增加：空头开仓推动，趋势持续性更强
价格下跌 + OI 减少：多头平仓推动，持续性弱一些
```

---

## 八、情绪层

### 1. 大户持仓多空比变化

```text
大户持仓多空比变化 =
    当前大户持仓多空比 - 上一周期大户持仓多空比
```

---

### 2. 全账户多空比变化

```text
全账户多空比变化 =
    当前全账户多空比 - 上一周期全账户多空比
```

---

### 3. 情绪判断

```text
如果 大户持仓多空比变化 > 0
并且 全账户多空比变化 < 0：
    情绪状态 = 大户偏多，散户偏空/离场

如果 大户持仓多空比变化 < 0
并且 全账户多空比变化 > 0：
    情绪状态 = 大户偏空，散户偏多/接盘

如果 大户持仓多空比变化 > 0
并且 全账户多空比变化 > 0：
    情绪状态 = 多头共识增强

如果 大户持仓多空比变化 < 0
并且 全账户多空比变化 < 0：
    情绪状态 = 空头共识增强

否则：
    情绪状态 = 中性
```

---

### 4. 情绪分

```text
如果 大户持仓多空比变化 > 0
并且 全账户多空比变化 < 0：
    情绪分 = +10

如果 大户持仓多空比变化 < 0
并且 全账户多空比变化 > 0：
    情绪分 = -10

如果 大户持仓多空比变化 > 0
并且 全账户多空比变化 > 0：
    情绪分 = +5

如果 大户持仓多空比变化 < 0
并且 全账户多空比变化 < 0：
    情绪分 = -5

否则：
    情绪分 = 0
```

---

## 九、风险层

### 1. 需要计算的风险指标

```text
ATR
资金费率
短周期涨跌幅
强平金额，可选
```

---

### 2. 风险扣分

```text
风险扣分 = 0
```

```text
如果 准备做多 且 资金费率极高：
    风险扣分 -= 10
```

```text
如果 准备做空 且 资金费率极低：
    风险扣分 -= 10
```

```text
如果 短周期涨跌幅过大：
    风险扣分 -= 10
```

```text
如果 ATR 过大：
    风险扣分 -= 10
```

```text
如果 刚发生大量强平，且准备追同方向：
    风险扣分 -= 10
```

风险扣分最多：

```text
风险扣分 >= -30
```

---

### 3. 风险等级

```text
如果 风险扣分 >= -10：
    风险等级 = 低

如果 风险扣分 < -10 且 风险扣分 >= -20：
    风险等级 = 中

如果 风险扣分 < -20：
    风险等级 = 高
```

## 十、总分模型

```text
总分 =
    趋势分
  + 动量分
  + 仓位分
  + 情绪分
  + 风险扣分
```

总分范围大致：

```text
-100 到 +100
```

---

## 十一、交易信号规则

```text
如果 总分 >= 60：
    交易信号 = 强多
    操作建议 = 只找回踩做多，不追高

如果 总分 >= 30 且 总分 < 60：
    交易信号 = 偏多
    操作建议 = 轻仓做多，或等待回踩确认

如果 总分 > -30 且 总分 < 30：
    交易信号 = 震荡
    操作建议 = 不做，或只做区间

如果 总分 <= -30 且 总分 > -60：
    交易信号 = 偏空
    操作建议 = 轻仓做空，或等待反抽确认

如果 总分 <= -60：
    交易信号 = 强空
    操作建议 = 只找反抽做空，不追空
```

## 十二、特殊结构标签

### 1. 顺势多

```text
如果 当前价格 > EMA20
并且 EMA20 > EMA60
并且 主动买卖压力比例 > 10%
并且 成交额放大倍数 >= 1.2
并且 持仓价值变化额 > 0
并且 资金费率不过热：

    标签 = 顺势多
```

---

### 2. 顺势空

```text
如果 当前价格 < EMA20
并且 EMA20 < EMA60
并且 主动买卖压力比例 < -10%
并且 成交额放大倍数 >= 1.2
并且 持仓价值变化额 > 0
并且 资金费率不过度为负：

    标签 = 顺势空
```

---

### 3. 疑似吸筹

```text
如果 价格横盘
并且 持仓变化比例 < 5%
并且 大户持仓多空比变化 > 0
并且 全账户多空比变化 < 0
并且 主动买卖压力比例 <= 0
并且 价格不跌：

    标签 = 疑似吸筹
    操作建议 = 不立刻追，等待向上突破确认
```

解释：

```text
大户偏多增强
全账户偏弱
主动卖出压力存在但价格不跌
可能是下方承接或主力吸筹
```

---

### 4. 疑似派发

```text
如果 价格横盘
并且 持仓变化比例 < 5%
并且 大户持仓多空比变化 < 0
并且 全账户多空比变化 > 0
并且 主动买卖压力比例 >= 0
并且 价格不涨：

    标签 = 疑似派发
    操作建议 = 不立刻空，等待向下跌破确认
```

解释：

```text
大户偏多减弱或偏空增强
全账户偏多增强
主动买入压力存在但价格不涨
可能是上方压制或主力派发
```

## 十五、测试计划与可执行性保证

这不是建议流程，而是交付前的硬性验收标准。

### 1. Red 阶段

先补测试，再写实现。测试必须先失败，且失败原因要可解释、可定位。

必须覆盖：

```text
三类长短比数据是否纳入补齐链路
EMA20 / EMA60 / ATR 是否能从现有序列算出
总分与交易信号分档是否符合阈值
多交易所 OI 权重聚合是否正确
评分列表 API 是否返回可渲染快照
评分页面是否能正常打开并展示核心字段
```

### 2. Green 阶段

实现必须满足以下条件后才算通过：

```text
Python 单测全部通过
Playwright 契约测试全部通过
页面在本地浏览器中无 500 / 404 / 主要字段报错
刷新按钮能触发补齐与重新拉取
```

### 3. Refactor 阶段

只允许做不改变行为的整理：

```text
抽取评分计算函数
统一字段命名与兼容层
优化页面结构与展示
保留测试全部通过
```

### 4. 可执行性约束

为了避免“计划看起来完整，但落地不可做”，实现前必须先满足这些前置条件：

```text
接口字段已在本地模型中找到对应来源
缺失字段有兼容读取方案
评分页数据来源能在现有查询层拼出
每个新增接口都有至少一条失败测试先行
```

### 5. 最终验收

当且仅当下面条件同时满足，才认为本方案可交付：

```text
评分页能返回 200
评分 API 能返回排序后的快照
刷新操作能完成补齐并刷新评分
测试命令在本地连续运行通过
```

## 十六、交易纪律

模型不是为了每根 K 线都交易。

模型只做三件事：

```text
1. 过滤不该做的行情
2. 找高胜率方向
3. 控制仓位和止损
```

必须加硬规则：

```text
没有趋势，不做
总分不够，不做
止损太远，不做
资金费率太极端，不追
连续亏 2 次，当天停止
单笔风险不超过本金 1%
不要满仓
不要扛单
不要用模型解释亏损
```

## 十七、最终结论

不要追求：

```text
真实主力净流入
```

应该做：

```text
市场结构评分模型
```

核心是：

```text
趋势 + 主动买卖压力 + OI + 大户/全账户分歧 + 风险过滤
```

最终目标是输出：

```text
当前更适合做多、做空、观望，还是只等确认
```
